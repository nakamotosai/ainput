# ainput SPEC

## 1. 项目目标

从零开始设计并实现一个 Rust 主导的 Windows 本地语音输入工具 `ainput`。

它当前首版的目标不是翻译，也不是系统级输入法，而是：

- 让用户按住 `Ctrl+Win` 即可开始录音
- 松开 `Ctrl+Win` 后结束录音并触发本地离线 ASR
- 将识别结果直接插入当前 IDE / 浏览器 / 聊天输入框
- 当自动插入失败时，至少稳定写入剪贴板

## 2. 产品定位

- 产品名：`ainput`
- 类型：Windows 本地常驻工具
- 用户：项目作者本人
- 优先级：轻量、快速、稳定、自用效率优先

## 3. 技术路线

- 主语言：Rust
- ASR 模型：SenseVoiceSmall
- 推理接入：sherpa-onnx Rust API
- 兜底：sherpa-onnx C API
- 运行时：尽量只保留 Rust + sherpa-onnx 相关动态库

## 4. 当前设计口径

### 4.1 正式支持语种

- 中文
- 粤语
- 英语
- 日语
- 韩语

### 4.2 实际优化重点

- 中文口述输入
- 中英混合技术内容的基础识别可用性
- Windows 当前输入区域的直接粘贴体验

## 5. 本期必须项

- 用 Rust 重新设计项目骨架
- 建立可运行的 Cargo workspace
- 打通 `Ctrl+Win` 热键 -> 录音 -> ASR -> 文本输出主链路
- 实现自动粘贴与剪贴板输出
- 实现最小失败降级策略
- 启动后提供可见的托盘常驻入口
- 在按住录音期间提供屏幕下方、任务栏上方的底部悬浮条反馈
- 至少在一个 IDE 输入框和一个浏览器输入框中验证可用

## 6. 本期不做

- 系统级 IME 注册
- 在线大模型作为主链路
- 原项目 Python UI 兼容
- 跨平台适配
- 插件系统
- 多用户同步
- 自动生成提示词
- 术语增强第一版
- 模板 / 场景 / 账本系统

## 7. 关键约束

- 首版必须围绕自用效率，不追求大而全
- 最终运行时尽量不依赖 Python
- UI 必须极简
- 当前阶段不让后续增强能力阻塞主链路交付
- 所有任务推进默认回写本目录文档

## 8. 验收标准

### 8.1 功能验收

- 按住 `Ctrl+Win` 可启动录音
- 松开 `Ctrl+Win` 可停止录音并触发本地离线识别
- 识别结果可直接进入当前输入框
- 自动插入失败时可稳定进入剪贴板
- 启动后用户能在托盘确认程序仍在运行
- 按住录音时用户能看到屏幕下方、任务栏上方的底部悬浮条，松开后悬浮条消失
- 至少在一个 IDE 输入框和一个浏览器输入框里完成实测

### 8.2 架构验收

- Rust 为主程序语言
- sherpa-onnx 为 ASR 运行时核心
- Python 不进入最终常驻主链路
- 项目目录职责清晰
- 工作流、日志、任务清单齐全

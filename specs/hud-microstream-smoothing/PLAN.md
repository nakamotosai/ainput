# HUD 微流式吐字平滑 PLAN

> 状态：已被 `../streaming-realtime-rewrite-v3/` 取代。本文只保留历史背景；继续开发和验收以 V3 `SPEC.md` / `PLAN.md` / `TASKLIST.md` 为准。

## 目标

- 让 HUD 的出字体感从“块状跳字”变成“连续逐字吐字”。
- 不破坏现有 `preview.9` 的识别稳定性和快速上屏。

## Round 1

### 目标

- 在不改 worker 主链的前提下，让 HUD 支持目标文本和显示文本分离。

### 小任务

1. 在 `main.rs` 增加 HUD 逐字模式接线。
2. 在 `overlay.rs` 增加 `hud_target_text / hud_display_text`。
3. `StreamingPartial` 到来时不再直接整段替换 HUD。
4. `tick()` 中按固定节奏逐字推进 HUD 显示文本。

### 验收

- HUD 能持续逐字吐字，而不是收到 partial 时整段跳变。

## Round 2

### 目标

- 处理修正、回退和最终文本收尾。

### 小任务

1. 实现最长公共前缀 diff。
2. 对新增、删改、整段重写做分场景处理。
3. `StreamingFinal` 时立即对齐最终文本并停止动画。
4. 增加 HUD 微流式日志和纯逻辑测试。

### 验收

- 修正时 HUD 不闪、不抖、不倒退得很怪。
- 最终上屏速度不受影响。

## Round 3

### 目标

- 直接压短预览周期，不走渐进试探。

### 小任务

1. 把预览周期直接改成：
   - `0-3s = 120ms`
   - `3-8s = 220ms`
   - `8s+ = 360ms`
2. 跑编译和真包验证。
3. 交给真机直接体验验收。

### 验收

- HUD 的吐字明显更连续。
- 长句稳定性和最终上屏速度不回退。

# preview.5 真流式标点重构 PLAN

## Phase 1: 配置与规格回正

### 目标

- 把 preview.5 的执行基线从“规则补标点/伪流式优化”改成“真流式 ASR + 官方中文标点模型”。

### 动作

1. 重写任务级 `SPEC.md` / `PLAN.md`。
2. 统一确认候选模型目录名、体积和打包口径。
3. 识别当前代码里哪些地方仍把 `rewrite_enabled` 当作规则补标点开关。

### 验证

- 规格文件与用户最新目标一致。
- 模型与配置真相源明确。

## Phase 2: 接入官方中文标点模型

### 目标

- 在 Rust 侧增加可复用的中文离线标点封装。

### 动作

1. 在 `crates/ainput-asr/src/lib.rs` 增加：
   - `OfflinePunctuation` 配置结构
   - 模型发现逻辑
   - 创建与调用包装
2. 支持从模型目录中发现 `model.int8.onnx`。
3. 为非 ASCII 路径复用现有 runtime cache 逻辑。
4. 补标点层单测。

### 验证

- `cargo test -p ainput-asr` 相关测试通过。
- 标点模型目录缺失时能给出明确错误。

## Phase 3: 统一 HUD 与最终提交文本链

### 目标

- 让 HUD 与最终上屏使用同一条“最新一句加标点”链。

### 动作

1. 在 `worker.rs` 中把：
   - `prepare_streaming_preview_text`
   - `prepare_streaming_output_text`
   改成基于官方标点模型的统一处理。
2. 保留“只改最近一句”的状态机边界，不全文重写。
3. 让 preview 与 final 的文本准备共享同一套函数。
4. 规则补标点降级为非主路径或彻底移除其在 streaming 主链中的使用。
5. 保持“标点失败可回退到原文，但 HUD/最终仍同链”。

### 验证

- `worker.rs` 相关测试通过。
- 现有 streaming state 回归测试继续通过。
- 新增“HUD 与 final 同链”的测试通过。

## Phase 4: 模型切换与打包

### 目标

- 把流式模型切到 `sherpa-onnx-streaming-zipformer-zh-int8-2025-06-30`，并将标点模型纳入打包。

### 动作

1. 下载并落库流式中文模型。
2. 下载并落库中文标点模型。
3. 更新默认配置：
   - 流式 ASR 模型目录
   - 标点模型目录
   - 相关开关默认值
4. 更新 `package-release.ps1` 的复制与清单逻辑。
5. 清理老的目录名漂移。

### 验证

- 模型目录在仓库内可发现。
- 打包脚本能正确包含 preview.5 所需模型。

## Phase 5: 回归验证

### 目标

- 证明这轮改动至少在代码层和最小运行层闭环。

### 动作

1. 跑相关单测。
2. 跑最小构建验证。
3. 若 Linux 全量构建仍被已知跨平台依赖卡住，至少完成：
   - 相关 crate 测试
   - 关键模块编译
4. 记录未覆盖边界。

### 验证

- 输出明确的通过项、未覆盖项、阻塞项。

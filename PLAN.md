# ainput PLAN

## 阶段 0：项目启动

目标：
把项目起步所需的目录、规则、决策、计划和日志体系搭好。

输出：

- Cargo workspace 骨架
- AGENTS / SPEC / PLAN / TASKLIST / ARCHITECTURE / DECISIONS / WORKFLOW / OPLOG

验证：

- 新会话进入目录后可直接开始开发

## 阶段 1：Rust 基础骨架

目标：
建立 Rust workspace 与首版所需模块边界，不写完整业务，只先搭载体。

动作：

- 建立 workspace
- 建立主应用 crate
- 建立 shell / audio / asr / rewrite / output / data crate
- 建立配置与日志基础设施

验证：

- `cargo check` 可跑通最小骨架

## 阶段 2：ASR 主链路

目标：
打通 Rust -> sherpa-onnx -> SenseVoiceSmall 的最小识别链路。

动作：

- 选定 Rust API 或 C API 实际落点
- 设计模型目录约定
- 先建立 wav 文件到文本的最小闭环
- 再建立麦克风录音到文本的最小闭环

验证：

- 录音后可得到文本结果

## 阶段 3：输入控制与输出注入

目标：
实现一个能真正服务日常使用的最小常驻语音输入工具。

动作：

- `Ctrl+Win` 全局热键
- 录音状态机
- 剪贴板
- 自动粘贴
- 插入失败降级

验证：

- 至少在一个 IDE 和一个浏览器输入框中可用

## 阶段 4：极简 UI 与托盘

目标：
只保留必要交互，不做大窗体产品。

动作：

- 托盘
- 录音状态反馈
- 最近结果预览
- 最小设置入口

验证：

- UI 不干扰主流程

## 阶段 5：后续增强能力

目标：
在主链路稳定后，再逐步补强识别后的文本质量与复用能力。

动作：

- 术语增强
- 提示词转换
- 高频场景模板
- 账本复用

验证：

- 主链路稳定后再进入单项增强验收

## 阶段 6：打包与回归

目标：
形成可长期自用的可运行版本。

动作：

- 打包
- 模型部署说明
- 回归测试
- 操作日志回写

验证：

- 本机可持续日用
